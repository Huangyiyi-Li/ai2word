# 需求文档：修复导出 Word 标题蓝色问题

## 版本
v202603191200

## 问题描述
导出的 Word 文件中，标题（H1-H6）文字显示为蓝色而非黑色。

## 根因分析
1. **docx.js 库内置了 Word 默认样式**：`styles.xml` 中的 Heading 样式包含 `themeColor` 属性引用（如 `accent1`），Word 渲染时会将这些引用解析为蓝色。
2. **theme1.xml 中的 accent 颜色为蓝色**：Word 的默认主题中 `accent1` 等颜色值为蓝色系。
3. **之前的修复方案不够全面**：正则表达式假设了固定的 XML 属性顺序（`w:val` 在前、`w:themeColor` 在后），未覆盖所有可能的属性排列。

## 解决方案

### 修改文件
- `webapp/lib/converter.js`
- `extension/lib/converter.js`

### 修改内容（postProcessDocx 函数中的标题颜色修复逻辑）

#### 步骤1：全局清理 themeColor
使用通配匹配 `[^>]*?`，覆盖任意属性排列顺序的 `<w:color ... w:themeColor="..." ... />` 标签。

#### 步骤2：定位 Heading 样式块
精确匹配 `styles.xml` 中 `styleId` 为 `Heading1`~`Heading6`、`Title`、`Subtitle` 的样式块，将其中所有 `<w:color>` 标签强制替换为黑色。

#### 步骤3：覆盖继承样式
匹配 `basedOn` 为 Heading 的派生样式，同样强制黑色。

#### 步骤4：蓝色系安全网
对 `styles.xml` 中所有 `<w:color w:val="XXXXXX"/>` 标签，通过 RGB 分量判断是否为蓝色系（蓝色分量高于红色和绿色且超过阈值），如果是则替换为黑色。

#### 步骤5：theme1.xml 修复
将 `accent1` 和 `accent5` 的 `srgbClr` 值替换为 `000000`。

#### 步骤6：document.xml 清理
清除 `document.xml` 中所有携带 `themeColor` 属性的 `<w:color>` 标签。

## 测试方法
1. 在 webapp 或 extension 中输入包含多级标题的 Markdown 内容
2. 导出为 Word 文件
3. 用 Microsoft Word 打开，验证所有标题文字为黑色
4. 确认正文等其他内容的颜色不受影响
