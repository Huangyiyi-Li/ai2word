# 内容搜索功能需求文档 (PRD)

## 背景
用户反馈：大模型对话内容很长，难以快速找到需要修改/导出的部分。

## 核心洞察
- ❌ 导出图片/PDF → 无法编辑，意义不大
- ✅ **内容搜索+定位** → 快速找到目标内容，再导出编辑 → **高价值**

## 功能需求

### 1. 网页端搜索 (webapp)
**场景**：用户粘贴对话内容后，快速定位到特定段落

**功能**：
- [ ] 实时搜索框（支持正则/普通搜索）
- [ ] 高亮匹配内容
- [ ] 跳转到上一个/下一个匹配
- [ ] 显示匹配数量 (3/10)
- [ ] 支持大小写敏感切换

**UI位置**：
```
输入框上方工具栏
[搜索 🔍] [替换] [大小写] [正则]  共找到 5 处
```

### 2. 插件端搜索 (extension)
**场景**：在 ChatGPT/DeepSeek 等页面，搜索历史对话中的内容

**功能**：
- [ ] 悬浮搜索按钮
- [ ] 搜索面板（覆盖层）
- [ ] 搜索当前对话历史
- [ ] 跳转到匹配位置
- [ ] 一键导出该部分内容

**实现方式**：
- 注入搜索 UI 到对话页面
- 遍历 DOM 查找对话内容
- 滚动定位到匹配项
